# IM2PPT

IM2PPT 是一个面向办公协作场景的 AI Agent 原型项目，目标是打通从 IM 群沟通到演示文稿生成的核心链路，减少信息整理与汇报制作的人力成本。

项目围绕智能办公落地能力设计：
- 从群聊中提取真实讨论内容
- 用大模型生成结构化汇报信息
- 自动产出并回传可用的演示文件
- 以模块化方式支持后续扩展到文档、模板、记忆等能力

## 当前已实现功能

1. 飞书事件回调接入
- 支持 challenge 验证
- 支持群消息触发与任务分发

2. 群聊内容采集与清洗
- 拉取最近 30 分钟群消息
- 过滤系统消息、机器人消息和无效触发文本
- 生成可用于模型输入的上下文

3. AI 大纲生成与结构化计划
- 调用 SiliconFlow 模型生成 PPT 大纲
- 将大纲转换为可渲染的结构化计划

4. PPT 文件生成与回传
- 使用 python-pptx 生成本地 .pptx 文件
- 上传飞书文件并在群内发送文件消息

5. 稳定性基础能力
- 回调事件幂等去重，避免重复执行
- 同群任务并发保护，避免同一群重复跑多次
- 回调快速返回 + 后台处理，降低平台重试概率

## 端到端流程

@机器人触发 -> 读取群聊上下文 -> AI 生成大纲 -> 结构化计划 -> 生成 PPT -> 上传飞书 -> 回传群聊

## 项目结构

- app.py: Web 回调入口、流程编排、幂等与并发保护
- bot/: 飞书接口封装与消息解析
- agents/: 模型调用与大纲/计划生成
- render/: PPT 渲染能力
- schemas/: 结构化数据定义
- temp/: 临时文件输出目录

## 运行方式

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 配置环境变量

```bash
copy .env.example .env
```

最少需要配置：
- FEISHU_APP_ID
- FEISHU_APP_SECRET
- SILICONFLOW_API_KEY

3. 启动服务

```bash
python app.py
```

4. 联调说明
- 飞书事件订阅地址指向服务的 /callback
- 确保已开通文件上传相关权限（如 im:resource:upload 或 im:resource）
- 在群内 @机器人并输入生成指令，机器人会回传 .pptx

## 适用价值

该原型证明了在办公协作中，通过 AI Agent 串联 IM 与演示制作流程，可显著缩短从讨论到汇报交付的周期，为多端协同和模块化场景编排提供可复用基础。
